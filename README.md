# 🏗️ AI BuildGraph Precision Pro

https://buildgraphai-cjpdngwlnv4gvu2frjbthh.streamlit.app/

**2D CAD 도면(DXF) 시맨틱 파싱 및 공간 지식 그래프(GraphRAG) 추출 엔진**

본 프로젝트는 복잡한 비정형 건축 CAD 도면(.dxf)을 업로드받아, AI(LLM) 기반의 시맨틱 텍스트 분석과 기하학적(Geometry) 위상 연산을 통해 도면 내 공간(Room)을 자동으로 구획합니다. 분할된 공간은 정밀한 면적, 축척, 인접성(Connectivity) 정보가 포함된 **지식 그래프(Knowledge Graph) JSON 데이터**로 변환되어, 다중 에이전트 논리 추론(예: Mafia Engine)을 위한 고정밀 공간 환경 데이터로 활용됩니다.

---

## ✨ 핵심 아키텍처 및 주요 기능 (Core Features)

### 1. 지능형 시맨틱 파싱 및 무장애 파이프라인 (AI & Fallback)
* **LLM 기반 텍스트 분류:** 도면 내 모든 텍스트를 추출하여 `gpt-4o-mini` 모델을 통해 `ROOM`, `FIXTURE`, `DIMENSION`으로 자동 분류합니다. "BEDROOM1 10'x11'"처럼 이름과 숫자가 혼재된 비정형 텍스트도 문맥을 파악해 정확히 방(ROOM)으로 식별합니다.
* **자체 Fallback 시스템:** API 응답 지연이나 형태 오류(JSON 파싱 에러) 발생 시, 시스템이 중단되지 않고 즉시 정규식 및 키워드 기반의 하드코딩 룰셋 엔진으로 전환되어 안정적인 분석(Zero-downtime)을 보장합니다.

### 2. 정밀 기하학 및 도면 블록(Block) 분해 로직
* **가상 엔티티 분해 (Virtual Entities):** CAD 특유의 그룹화된 블록(`INSERT` 타입) 객체를 내부 선분 단위로 완벽히 분해하여 숨겨진 기하학 데이터를 추출합니다.
* **부모 레이어 상속 (Layer Inheritance):** 블록 내부 선분의 레이어가 기본값(`0`)으로 지정되어 타입이 덮어씌워지는 CAD 도면의 고질적인 버그를 방지하기 위해, 부모 객체의 레이어 속성을 상속받도록 처리했습니다.

### 3. 스마트 레이어 마스터링 및 가구(Fixture) 배제
* **시맨틱-기하학 동기화:** LLM이 특정 텍스트를 가구(`FIXTURE`)로 분류하면, 해당 텍스트가 속한 레이어 전체의 기하 객체를 가구로 강제 전환합니다.
* **장벽(Barrier) 최적화:** 가구로 판명된 객체들은 공간을 나누는 물리적 장벽 연산에서 철저히 배제되어, 방의 면적(Polygon)이 가구의 모양대로 파편화되거나 구멍이 뚫리는 현상을 원천 차단합니다.

### 4. 시각/논리 분리형 문(Door) 처리 알고리즘
* **모양 기반 1/4원(Arc) 감지:** 레이어 이름에 의존하지 않고, 수학적으로 약 90도(70°~110°)를 그리는 원호를 탐지하여 문(Door)으로 강제 인식합니다.
* **Arc-to-L-Shape 변환:** 문이 차지하는 궤적 때문에 방의 코너 면적이 깎이는 것을 막기 위해 시스템 내부적으로는 문을 **시작점 $\rightarrow$ 중심점(Hinge) $\rightarrow$ 끝점을 잇는 직각(L자) 장벽**으로 변환하여 완벽히 밀폐합니다. 단, 사용자 UI에는 원래의 1/4원 곡선을 유지하여 가독성을 높였습니다.

### 5. 위상 수학 기반 공간 구획 및 노이즈 격리
* **Ray-casting & Polygon Union:** 식별된 벽과 문 선분들을 결합(Union)하고 팽창(Buffer)시켜 완벽히 닫힌 다각형(Closed Polygon) 공간들을 추출합니다. 열린 공간(예: 거실과 주방)에 여러 텍스트가 존재할 경우 이를 스마트하게 하나의 공용 공간으로 병합합니다.
* **Zero-Threshold 노이즈 격리:** 도면상의 미세한 틈새나 벽체 내부 빈 공간(Wall Cavity)을 주변 방에 무분별하게 합치지 않고 `UNKNOWN_SPACE`로 분리 및 은닉시킵니다. 이를 통해 추출되는 노드(Node) 데이터의 순도를 100%로 끌어올렸습니다.

### 6. 공간 연결성(Connectivity) 추출 및 JSON Export
* 물리적으로 인접한(5.0 CAD Unit 이내) 방과 방 사이의 교차점(Intersection)을 연산하여, 어떤 방들이 서로 연결되어 이동 가능한지(Edge)를 도출합니다.
* 분석된 모든 데이터는 고차원 추론 에이전트가 즉시 사용할 수 있도록 `spatial_graph.json` 형태로 Export 됩니다.

### 7. 동적 스케일 캘리브레이션 (Dynamic Scale Calibration)
* 도면 텍스트에 표기된 실제 치수(Explicit)와, 생성된 폴리곤 기반의 시스템 계산 치수(Estimated)를 비교 분석하여 도면의 현실 축척(Scale)을 도출합니다.
* 오픈 공간(주방, 거실 등)을 배제하고 밀폐된 공간만을 기준으로 삼으며, 사용자가 사이드바에서 기준 방을 변경할 때마다 즉각적으로 전체 도면의 실면적(sqft)이 재계산되는 인터랙티브 UI를 제공합니다.

## 🕵️‍♂️ 딥 다이브 추론 로그 해석 가이드 (Log Interpretation)

시스템 하단의 **'딥 다이브 추론 과정 디버거'**를 통해 엔진의 연산 과정을 투명하게 추적할 수 있습니다.

### 1. 텍스트 분류 및 예외 객체 필터링
> `🧠 [LLM 분류 성공] 28개 단어 식별 (9.9초)`
> `⚠️ [LINE_46] 기하 추출 오류: unsupported DXF type: DIMENSION`
* **해석:** LLM이 도면 내 텍스트의 성격을 성공적으로 파악했습니다. `DIMENSION`(치수선)이나 `MLEADER`(지시선) 같은 비물리적 객체는 장벽 연산에서 의도적으로 무시(Pass)하여 공간 계산의 노이즈를 방지합니다.

### 2. 기하학적 문(Door) 감지 및 보정
> `🚪 [LINE_118] 문 감지 (각도차: 90.0도) -> 직각(L자) 밀폐 장벽 생성`
* **해석:** 시맨틱 정보 없이 기하학적 형태만으로 문을 자동 식별했습니다. 방 면적의 오차를 없애기 위해 내부적으로 직각(L자) 장벽을 세워 공간을 밀폐한 과정을 나타냅니다.

### 3. 노이즈 격리 및 독립 보존 (Zero-Threshold)
> `🧱 [독립 보존] 16 면적 -> UNKNOWN_SPACE 로 분리`
> `🧱 [독립 보존] 2653 면적 -> UNKNOWN_SPACE 로 분리`
* **해석:** 면적이 16인 미세한 선 틈새나 2653인 벽체 내부 빈 공간(Wall Cavities)을 실제 방에 억지로 편입시키지 않고 `UNKNOWN_SPACE`로 분리했습니다. 오직 유의미한 공간만이 지식 그래프에 남게 됩니다.

## 🚀 How to Run (실행 방법)

1. Python 환경(3.8 이상 권장)에서 필요한 패키지를 설치합니다.
   ```bash
   pip install -r requirements.txt
   ```
2. 프로젝트 루트 경로에 .env 파일을 생성하고 OpenAI API 키를 입력합니다.
   ```bash
   OPENAI_API_KEY="sk-..."
   ```
3. Streamlit 앱을 실행합니다.
   ```bash
   streamlit run app.py
   ```
